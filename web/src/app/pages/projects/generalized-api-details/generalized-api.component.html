Generalized API Framework

Last Updated Mar. 2025

Overview

This project is a sample API for accessing a SQL database, created with the goal of using principles of generalization and inheritance to implement the base CRUD operations for any type of SQL table. This framework aims to use as much shared functionality as possible to ensure a well-organized codebase and make it as easy as possible to add new routes and database objects.

In addition to providing base CRUD operations on SQL objects including filtering and sorting, this framework also implements automatically generated Swagger documentation for each API route, including a title, description, and the response structure.

The source code for this project can be found on my GitHub page. Instructions for running locally are included in the README in the root directory.

Project Highlights

  • Generalized base classes for handling CRUD operations on SQL tables.
  • Expanded base classes for generating routes to handle one-to-many and many-to-many relationships.
  • Generalized flask routing templates to create routes needed for each CRUD and relationship operation.
  • Built-in swagger documentation.
  • API and SQL database containerized with docker.
